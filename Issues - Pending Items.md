16. **Site — journey page auto-link is "first occurrence per RENDERED SEGMENT" not "first occurrence per page"** (low / known quirk; design-acceptable).
    Both `foundations.astro` and `day-1.astro` segment the markdown body on `## Step N — Title` boundaries and call `processor.render(step.bodyMd)` separately for each step. The remark plugin's first-occurrence tracking lives inside one `runTransform` call — each segment gets a fresh `matched: Set<string>`. So a term like "Claude Code" can be wrapped once per step (i.e. 6 times across the foundations page's 6 steps), not just once on the whole page.
    **Design verdict:** acceptable. Readers often land on a specific step out of context (deep link from a search result or the in-page anchor nav) — having the link present in each step is arguably *more* helpful than less. AC7 stated "first occurrence per page" but the spirit is "don't be visually noisy"; one button per step body is still single-occurrence in the reader's local context.
    **If we want strict per-page first-occurrence:** refactor the journey pages to render all segments in a single processor call (concat with separator marker, render once, then split the resulting HTML on the marker). Adds complexity for a small visual win. Not doing this now.
